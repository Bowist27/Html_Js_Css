¿Cuál es la diferencia entre Internet y la World Wide Web? En que el internet es el que permite la comunicacion e intercambio de informacion y la World Wide Web es el sistema que funciona a traves del internet mismo para acceder y viusalizar informacion a traves de los navegadores web
¿Cuáles son las partes de un URL? Protocolo(Metodo de acceso HTTP, HTTPS), Subdominio (subdivision del dominio principal como www), Dominio (Direccion del servidor donde se encuentra un recurso), TLD (extension de la ruta como .com u .org), Ruta(Ubicacion especifica del recurso en el server como /file.html), Parametros y fragmento (Variables que pasan al recurso y pueden modificar el comportamiento, e indican una ubicacion especifica en el recurso)
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PATCH, PUT, DELETE?. Para poder utilizar recursos e informacion que se encuentre en un servidor, basicamente el hacer una accion en internet, (GET: para solicitar datos al server), (HEAD:Solicita encabezados sin obtener el body), (POST:envia datos al server) (PUT: envia los datos al server para guardar en una ruta especifica), (PATCH: Modifica de forma parcial un recurso), (DELETE: elimina el recurso encontrado en el server)
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?. POST, porque envía datos de forma oculta sin embargo no de forma cifrada, permite un envio mayor de datos que GET
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? Metodo GET, esto porque no modifica el estado del server, y se usa para recuperar los recursos del servidor

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información? Ciclo de vida de los sistemas de información: Este describe etapas por las que pasa desde el concepto hasta su eliminación.
  1. Planificación: Se define la necesidad estableciendo objetivos y se realiza un estudio de factibilidad.
  2. Análisis: Se analiza una situación para definir los requisitos.
  3. Diseño: Se realiza la interfaz de usuario y las bases de datos.
  4. Desarrollo: Se implementa la codificación, pruebas y depuración.
  5. Implementación: Se instala el sistema y se capacita a los usuarios.
  6. Operación y mantenimiento: Se monitorea una alpha, se realizan correcciones de errores y se actualiza según sea necesario.
  7. Descontinuación: Se apaga el sistema cuando ya no cumple con las necesidades del cliente.

Tomar en cuenta si en verdad se necesita pues este se usa en ocasiones epeciales pues tiene un gran peso al poder saltar paso de prioridad entonces no tendrias porque modificar la jerarquia de todos los selects, el uso excesivo de este hace que el codigo sea mas dificil de mantener debido a que rompe por completo la cascada de estilos y por lo tanto el entendimiento de como se aplican las reglas a cada elemento ageregado

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt? Todo depende para que los vayas a usar:
  • % para elementos que necesitan ser adaptados a multiples tamaños de pantalla
  • px para tener algo en tamaño fijo especialmente imagenes
  • pt para tamaños fijos en léxico
¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio? Elimina caracteres inecesarios, todos los espacios en blanco ademas de los saltos de linea, si tienes un archivo mas pequeño se descargará más rápido en el cliente
